# Knowledge

Esta pasta guarda conhecimento persistente e reutilizável.

## O que deve entrar aqui

- decisões arquiteturais consolidadas;
- padrões do projeto;
- notas técnicas duráveis;
- snippets conceituais;
- aprendizados recorrentes.

## O que não deve entrar aqui

- tarefas passageiras;
- rascunhos operacionais do dia;
- documentos exclusivos de uma única feature.

## Objetivo

Reduzir perda de contexto e apoiar respostas com base em evidência do sistema.
