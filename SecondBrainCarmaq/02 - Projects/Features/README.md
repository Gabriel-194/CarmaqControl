# Features

Esta pasta centraliza o histórico de features do **CarmarqControl**.

Cada subpasta de feature deve conter os artefatos produzidos pelo fluxo de [[AI/Orchestrator]]:

- `PRD_<FEATURE>.md`
- `SPEC_<FEATURE>.md`
- `SECURITY_<FEATURE>.md`
- notas de implementação
- notas de QA
- notas de performance

## Convenção de nomenclatura

Use nomes curtos, descritivos e estáveis.

Exemplo:

- `cadastro-de-cliente/`
- `painel-financeiro/`
- `agenda-do-tecnico/`
